## 📝 リリースノート作成ガイド（社内向け）

### 対象読者
**エンドユーザー（薬局スタッフ・医療従事者）** に向けて、そのまま配布できる内容を作成します。

### 1. ファイル作成

該当製品・四半期のディレクトリに新しいリリースノートを作成します。

```bash
# 例：Kakeruの2025年Q3リリースノート作成
cp templates/release-note-template.md products/kakeru/releases/2025/Q3/v2.5.0_2025-08-27.md
```

### 2. 記載原則

#### ユーザー視点の徹底
- ✅ **良い例**: 「患者様の薬歴が見やすくなりました」
- ❌ **避ける例**: 「N+1問題を解消しました」

#### 専門用語の回避（メインコンテンツ）
- ✅ **良い例**: 「処理速度が向上しました」
- ❌ **避ける例**: 「PostgreSQL 17にアップグレードしました」
- 💡 **技術詳細は`<details>`タグ内に記載**

#### 具体的な操作手順
- ✅ **良い例**: 
  - 「1. 「患者メニュー」ボタンをクリックします」
  - 「2. 患者様を選択します」
  - 「3. 画面右下の青いボタンをクリックします」
  - 「1. 「管理ポータルへ」ボタンをクリックします」
  - 「2. 「文字列置換設定」カードをクリックします」
- ❌ **避ける例**: 
  - 「新機能を利用可能」
  - 「SOAP画面を開きます」（具体的な操作が不明）
  - 「管理画面から設定します」（ボタン名が不正確）

#### 技術詳細の記載方法
```markdown
### 機能名
ユーザー向けの説明文

<details>
<summary>🔧 技術詳細（クリックで展開）</summary>

- 実装PR: yakureki-front#123
- 技術的な変更内容
- パフォーマンス測定結果

</details>
```

## 📋 命名規則

### ファイル名
```
v{major}.{minor}.{patch}_{YYYY-MM-DD}.md
```

例：
- `v2.5.0_2025-08-27.md` - メジャーリリース
- `v2.4.1_2025-08-15.md` - パッチリリース
- `v2.4.0-hotfix1_2025-08-16.md` - ホットフィックス

## ✏️ 記述ルール

### 必須項目

1. **リリース情報**
   - バージョン番号
   - リリース日
   - 更新の概要（1-2文でユーザーメリットを説明）

2. **新機能・改善点**
   - ユーザーが実感できる変更のみ記載
   - 各項目に「お手元での確認方法」を必ず含める
   - **重要**: 確認方法は実際のUIに表示される文言を正確に使用
     - 例: 「1. 「患者メニュー」ボタンをクリックします」
     - 例: 「1. 「管理ポータルへ」ボタンをクリックします」
   - ボタンやメニュー名は必ず「」で囲む
   - スクリーンショットや図解を積極的に活用

3. **不具合の修正**
   - ユーザーが経験していた問題の解決を明記
   - 「〜できない問題を修正」という表現を使用

4. **お手元での確認方法**
   - 実際の業務フローに沿った確認手順
   - 「いつもの操作」を基準に説明
   - 新機能の場合は詳細な手順を記載

### 推奨カテゴリ（ユーザー向け表記）

- 🆕 **新しく追加された機能**
- 📈 **使いやすさの改善**
- ✅ **不具合の修正**
- ⚡ **処理速度の向上**
- 🔒 **セキュリティの強化**

### 技術情報の扱い方

#### メインコンテンツでは避ける
- データベースの変更
- フレームワークのアップデート  
- 内部アーキテクチャの変更
- PR番号やコミットハッシュ

#### `<details>`タグ内に含める
- PR番号とリンク
- 技術的な実装詳細
- パフォーマンス測定値
- 使用技術やライブラリ
- エラーコードや技術的な原因

**配布時の処理：**
- GitHub/GitLab: そのまま配布（折りたたみ表示）
- PDF/Word: `<details>`セクションを削除して生成
- メール: `<details>`セクションを削除してコピー

### 文体・トーン

- 敬語を使用（です・ます調）
- 親しみやすく、専門的すぎない表現
- ポジティブな表現を心がける
- 「改善しました」「便利になりました」など成果を強調

## 📄 テンプレート使用方法

`templates/user-release-note-template.md` を使用：

1. **ユーザー向け情報**
   - わかりやすい日本語で記載
   - 具体的な操作手順を含める
   - スクリーンショットを活用

2. **技術情報（`<details>`内）**
   - PR番号とリポジトリ名を明記
   - 実装の技術的詳細
   - パフォーマンス改善の数値

3. **配布方法別の対応**
   - **社内共有**: そのままMarkdownファイルを使用
   - **ユーザー配布**: `<details>`セクションを除去
